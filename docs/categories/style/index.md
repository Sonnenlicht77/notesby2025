# CSS



## CSS样式 [🔗](css/)



## 除了常规css,还有哪些特殊体系方案，详细说明各自的核心概念与优缺点

1. CSS
 - 传统css: 基于选择器和属性的样式定义，简单直接，适用于简单的样式需求。
 - 缺点： 复用性差，维护成本高

2. sass、less: 基于css的预处理语言，添加了变量、嵌套、Mixin等功能，提高了样式的复用性和维护性。
 - 优点： 提高了样式的复用性和维护性，减少了代码量;需要进一步封装，css hack (兼容性、厂商前缀);postcss 等工具可以进一步优化和处理css代码。
 - 缺点： 学习成本高，需要额外的工具支持

3. css modules: 基于css的模块化方案，每个组件都有自己的样式文件，避免了全局样式的冲突和维护问题。
 - 优点： 组件化开发，样式 scoped，避免了全局样式的冲突和维护问题。
 - 缺点： ts 类型支持不完善，定制化程度低

4. styled-components/utility-first css(tailwindcss): 
- 非常快，编译时生成样式，避免了运行时的性能问题
- 定制化程度高，可根据需求自定义样式

- 5. css-in-js(styled-components/emotion): 基于js的样式定义方案，将样式写在js文件中，避免了全局样式的冲突和维护问题。
 - 优点： 运行时生成样式，避免了编译时的性能问题; 定制化程度高，可根据需求自定义样式。
 - 缺点： 运行时增加了一定的性能开销，不适合对性能要求较高的场景。

 - 6. scope css: 基于css的作用域方案，每个组件都有自己的样式作用域，避免了全局样式的冲突和维护问题。
  - 优点：简单好理解, 避免了全局样式的冲突和维护问题。
  - 缺点：复用性差，维护成本高
